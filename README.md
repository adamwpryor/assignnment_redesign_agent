# AI-Resilient Course Modernization Pipeline

This repository contains an experimental, native "Domain Engine" for the **Gemini OS**. Its purpose is to perform zero-knowledge, automated pedagogical overhauls of legacy online course materials, transforming them into AI-resilient learning experiences.

## The Pedagogical Problem

Generative AI has fundamentally disrupted standard online course design. "Knowledge regurgitation" tasks—such as standard 5-paragraph essays, weekly discussion boards, or simple theoretical summaries—can now be bypassed entirely by an LLM with zero student comprehension.

If a grade is based 100% on a final text product, the assignment is vulnerable.

## The Solution: Process-Over-Product

This pipeline operates under a strict pedagogical constraint: **No assignment deliverable can be an unverified text document.**

Instead, the AI is positioned as a *required baseline tool*. The assignments generated by this pipeline force students to engage with the AI's output through highly contextualized, process-oriented verification methods.

Examples of these new deliverables include:

* **Visual Framework Defenses:** Generating an AI system map (e.g., Mermaid.js) and defending structural edits via an oral/video component.
* **"Artifact as Blueprint" Documentation:** Using AI to write foundational logic, but producing highly standardized technical documentation (like an Architectural Decision Record) that maps precisely to rigid institutional constraints.
* **In-Browser Iteration:** Recording a live screen-share demonstrating the real-time, iterative breakdown of a problem using an AI tool.
* **Hyper-Local Context Synthesis:** Integrating standard AI knowledge with specific, real-time local datasets (e.g., current university senate minutes) that the AI does not possess in its generalized pre-training data.

## System Architecture & Technical Implementation

This project is built directly into the Gemini OS ecosystem utilizing the **Local Context Hooking** protocol. It does not rely on fragile Python execution scripts; instead, it uses a purely declarative structure (JSON and Markdown) that plugs natively into the global Swarm Architecture.

### Directory Structure & Handoff Routing

The directory is split into the visible data folders (`input/` and `output/`), the master orchestrator (`GEMINI.md`), and the hidden execution environment (`.gemini/`).

```text
assignnment_redesign_agent/
│
├── GEMINI.md                    <-- Bootloader Orchestration 
│
├── .gemini/                     <-- Hidden Local Context Extension
│   ├── agents.json              <-- Local Agent Registry Hook
│   ├── skills.json              <-- Local Skill Registry Hook
│   ├── agents/
│   │   ├── vulnerability-assessor/persona.md
│   │   ├── resilient-designer/persona.md
│   │   └── blueprint-compiler/persona.md
│   │
│   └── skills/
│       └── local-context-fetcher/SKILL.md
│
├── input/
│   └── legacy_syllabus.md       <-- Initial sample data
│
└── output/                      <-- Destination for generated payloads
```

### The Orchestration Loop

This template utilizes three custom agent Specialists that pass structured JSON and Markdown payloads sequentially:

1. **Vulnerability Assessor (`.gemini/agents/vulnerability-assessor`)**:
    * **Input**: Reads `input/legacy_syllabus.md`.
    * **Logic**: Evaluates assignments using a "Predictability Matrix" (measuring Format Standardity, Context Availability, and Verification methods).
    * **Output**: Generates a structured `output/vulnerabilities.json` payload mapping vulnerable assignments to predictive AI-bypass pathways.
2. **Resilient Designer (`.gemini/agents/resilient-designer`)**:
    * **Input**: Reads `output/vulnerabilities.json`.
    * **Logic**: Uses the `local-context-fetcher` skill (a mocked Model Context Protocol server returning institutional data). It enforces the strict constraint to eliminate text deliverables, transforming them into process-driven tasks. Must *never* rely on "prompt logs."
    * **Output**: Generates `output/resilient_activities.md`.
3. **Blueprint Compiler (`.gemini/agents/blueprint-compiler`)**:
    * **Input**: Reads both the original `input/legacy_syllabus.md` and the new `output/resilient_activities.md`.
    * **Logic**: Acts as the auditor to ensure original learning objectives were preserved.
    * **Output**: Integrates everything into the final `output/modernized_course_blueprint.md`.

### Native Setup instructions

Because this project utilizes Gemini OS conventions, setup is minimal:

1. Ensure your global registry (`C:\Users\adamw\.gemini\registry\...`) is active.
2. Navigate the Gemini CLI/OS into this directory. The root `GEMINI.md` file will immediately hook the local `.gemini/agents.json` and `.gemini/skills.json` payloads into your active context.
3. You can execute the pipeline natively using your standard System Orchestrators.
